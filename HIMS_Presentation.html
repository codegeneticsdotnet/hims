Hospital Information Management System

Process Flow & System Architecture Analysis

High Level Process Flow

  • 1. Patient Registration (Entry Point)
  • 2. Clinical Assessment (OPD/IPD)
  • 3. Care & Treatment (Nursing/Meds)
  • 4. Ancillary Services (Lab/Pharmacy)
  • 5. Billing & Finance (Exit Point)

1. Patient Registration

  • Input: Patient Demographics (Name, Age, Address)
  • Action: Search existing or Create new record
  • Output: Unique Patient ID (PID)
  • Key Table: patient_personal_info
  • Roles: Receptionist, Clinic Staff

2. Out-Patient (OPD) Flow

  • Visit: Walk-in or Appointment
  • Action: Doctor Consultation
  • Data: Vitals, Diagnosis, Complaints
  • Orders: Prescriptions, Lab Tests
  • Disposition: Home or Admit (IPD)
  • Key Tables: patient_details_iop, iop_diagnosis

3. In-Patient (IPD) Flow

  • Admission: Room/Ward Assignment
  • Care: Daily Nurse Notes, Vitals Monitoring
  • Procedures: Bedside (Oxygen, Nebulizer)
  • Transfers: Tracking movement for billing
  • Key Tables: iop_room_transfer, iop_nurse_notes

4. Ancillary Services

  • Laboratory: Request -> Result Entry -> Verify
  • Pharmacy: Prescription -> Stock Check -> Dispense
  • Inventory: Real-time deduction (pharmacy_sales)
  • Key Tables: lab_service_request, pharmacy_inventory_details

5. Billing & Discharge

  • Room Charge: Days x Room Rate (from transfers)
  • Doctor Fee: From Discharge Advice
  • Items: Meds + Labs + Services
  • Process: Payment -> Receipt -> Discharge
  • Key Tables: iop_billing, iop_receipt

Database Analysis: Unused Tables

  • Legacy Inventory (o_*): o_batch, o_items, o_inventory
  • Empty Functional Tables:
  • - cart_billing, category_list
  • - doctors_fee (Replaced by discharge advice)
  • - patient_appointment
  • - surgical_package
